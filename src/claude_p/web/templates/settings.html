claude.ai usage integration experimental

The rate_limit_event surfaced in the ledger is a free signal from claude -p. It tells you when windows reset but not how much you've burned. For actual % utilization, claude-p can poll the (undocumented) claude.ai/api/organizations/<org>/usage endpoint on your behalf. This requires you to paste your live sessionKey cookie. Anthropic can break or remove this endpoint at any time — consider it best-effort.

Visit claude.ai/settings/usage in your browser. The UUID in the URL of the /api/organizations/<this>/usage request (DevTools → Network) is your org ID.
Copy the value of the sessionKey cookie from your claude.ai browser session. Stored in SQLite; rotated when you log out of claude.ai.
Only set this if you see 403 errors. Cloudflare rotates it ~hourly, so the integration will break until you paste a fresh one.
